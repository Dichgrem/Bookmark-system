# 一、基础评分（60 分）

1. **架构纯洁 - 10 分**：严格遵循前后端彻底分离，严禁出现 JSP/Thymeleaf。
2. **分层规范 - 10 分**：要求但不限于后端严格落地标准的系统架构，代码职责清晰。
3. **RESTful API 与交互 - 15 分**：要求但不限于至少 6 个规范的 RESTful 接口，前后端数据采用 JSON 格式对接，包含规范的统一返回体（Result/Response 对象）。
4. **数据持久化 - 15 分**：要求但不限于使用 PostgreSQL、Mysql、Firebase 等技术进行数据持久化，至少包含 4 张以上的核心业务表，具备一对多、多对多的关联查询处理。
5. **工程化基建 - 10 分**：要求但不限于使用 Maven、Gradle 进行管理依赖，且必须提交包含清晰历史记录的 Git 仓库，并记录团队协作文档。

> **🌟 小团队加成说明**：
> 如果小团队（2-4 人）完成整个项目（包含前端、后端、数据库及所有部署与文档），最终总分将视项目完成度、难度乘以 1.2 ~ 1.5 的小团队项目系数分。
> **最终得分 = (基础分 + 进阶分) × 项目系数分。**

# 二、进阶加分项（上限 60 分）

> **💡 说明**：自选组合技术点，并在期末展示中体现，加满 60 分为止（总分不超过 120 分）。

## 1. 高并发与分布式中间件（上限 40 分）

- **10 分**：包括但不限于引入 Redis 等缓存技术，将会话数据、高频字典表加载到内存，显著降低数据库压力。
- **15 分**：包括但不限于引入 RabbitMQ / Kafka / RocketMQ 等异步消息队列技术，将耗时操作（如发送邮件、生成长报告）转为异步消息队列处理。
- **20 分**：包括但不限于基于 Redis 或 ZooKeeper 进行并发控制：在核心交易环节（如扣库存、抢单）成功实现。

## 2. 接入多模态大模型技术（上限 40 分）

- **10 分**：包括但不限于成功接入国内外第三方大模型的简单 API，实现简单的文本摘要或翻译辅助等功能。
- **20 分**：包括但不限于运用简单的 Spring AI 或 LangChain4j 技术、或使用 Dify、Coze 等搭建工作流接入。
- **20 分**：包括但不限于使用私有知识库 RAG 技术：结合向量数据库实现私有知识库增强检索（RAG），或利用多模态模型完成简单的图像解析与逻辑推理。

## 3. 云端服务架构部署技术（上限 40 分）

- **10 分**：包括但不限于使用 Docker 技术进行容器化部署，实现云端前端、后端、数据库的一键丝滑启动。
- **20 分**：包括但不限于将单体应用至少拆分为 3 个独立的微服务，并使用 Nacos/Consul 进行简单的服务注册与发现。
- **20 分**：包括但不限于引入 Spring Cloud 等网关管理技术，实现统一的路由转发、跨域处理，并配置完成接口限流。

## 4. 服务端开发工程素养（上限 20 分）

- **10 分**：包括但不限于实现基于角色的访问控制，使用 Spring Security 技术或配合其他鉴权技术。
- **10 分**：包括但不限于使用一定的 AI coding 技术，并拥有按周期的固定 git 团队的协作记录，注解规范、API 接口文档。
